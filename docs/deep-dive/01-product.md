# 01. Что такое Dayflow

Dayflow — это backend для личного планировщика. Его цель — хранить разные типы информации о дне и превращать их в один практичный список **Today**: что действительно имеет смысл делать сегодня.

## Почему не просто список задач

Обычный TODO-лист показывает всё подряд. Dayflow учитывает контекст:

- есть ли дедлайн сегодня или он уже просрочен;
- была ли задача запланирована именно на сегодня;
- есть ли обязательные события;
- какие повторяющиеся блоки расписания попадают на текущий день недели;
- здоров ли пользователь;
- сколько у пользователя энергии;
- какую энергию требует задача;
- какие задачи уже выполнены, пропущены или архивированы.

## Главная идея продукта

Система хранит сырые данные:

- tasks — задачи;
- events — календарные события;
- schedule blocks — повторяющиеся блоки недели;
- day state — состояние дня;

а затем собирает из них dashboard:

```json
{
  "date": "2026-05-14",
  "dayState": {},
  "mandatoryEvents": [],
  "plannedEvents": [],
  "scheduleBlocks": [],
  "deadlineTasks": [],
  "plannedTasks": [],
  "suggestedTasks": []
}
```

## Для каких клиентов готовится backend

Backend специально не привязан к одному интерфейсу. Его можно использовать из:

- Web/PWA клиента;
- Telegram bot;
- Desktop app;
- Cloudflare Tunnel для доступа извне;
- любых скриптов через HTTP API.

## Что важно сохранить при развитии

- HTTP-логика должна жить в routes.
- Бизнес-логика должна жить в services.
- Работа с базой должна жить в repositories.
- Преобразование DB-моделей в API-ответы должно жить в mappers.
- Today logic должна оставаться разбитой на маленькие функции, потому что правила будут меняться чаще всего.
